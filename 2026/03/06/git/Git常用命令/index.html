Git常用命令

Git常用命令流程图

各区域含义：

  • Remote（远程库）：GitHub/GitLab/服务器上的仓库。
  • Repository（本地库）：.git目录；保存完整历史和分支。
  • Index（暂存区/staging area）：提交前的缓冲区。
  • Workspace（工作区）：实际编辑的文件所在目录。

配置

Git配置级别：
系统级 (–system)：/etc/gitconfig，所有用户生效
全局级 (–global)：~/.gitconfig，当前用户生效
仓库级 (–local)：.git/config，当前项目仓库生效，默认为local

配置优先级： local > global > system

分支与标签

branch（分支）是可变、动态的开发线，用于功能开发、协同工作和代码合并，随提交向前移动。
tag（标签）是不可变、静态的特定提交快照，用于标记发布版本或里程碑，一旦创建不再改变。

merge和rebase的差别：

慎用rebase，可以在本地私人分支上使用，但千万不要在多人合作的分支上使用rebase！

switch：只管“切分支、建分支、游离HEAD”
restore：只管“恢复文件内容、丢弃修改”
checkout：把这俩都混在一起，现在官方推荐使用上面两条新命令，更清晰

同步

fetch：只“下载”（origin/*），不合并
pull：“下载”并合并到本地相应分支，等同于fetch+merge
