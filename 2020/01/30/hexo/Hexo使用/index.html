Hexo使用

2020年01月30日 留言

Hexo是一款基于Node.js的快速、简洁且高效的静态博客框架。通过解析markdown文章，可在几秒内生成静态网页，拥有丰富的主题和插件。主要用于搭建个人博客，并支持一键部署到github等平台。

安装

全局安装hexo-cli

$ npm install -g hexo-cli

安装后可以直接使用hexo <command>

当前工作目录下安装hexo

$ npm install hexo

安装后可以通过命令npx hexo <command>或./node_modules/.bin/hexo <command>使用

ps:hexo-cli(command line interface)应该是在hexo的基础上封装了些命令行（shell,bat,ps1）

常用命令行

init

$ hexo init [folder]

在folder下新建一个网站。若无folder,则在当前文件夹。

主要执行两步：

  1. Git clone hexo-starter。
  2. 使用Yarn 1、pnpm或npm包管理器下载依赖（如有已安装多个，则列在前面的优先）。

目录结构：

new

$ hexo new [layout]  <title>

按布局新建文章或页面，默认为post，可以修改_config.xml中的default_layout来设置默认布局。

generate

$ hexo generate

生成静态文件

server

$ hexo server

启动服务器，默认预览地址为：http://localhost:4000/

depoly

$ hexo depoly

部署网站

clean

$ hexo clean

清除缓存

config

$ hexo config [key] [value]

列出网站配置_config.yml。只指定key则显示，均指定则赋值。

官方指令文档,详见https://hexo.io/zh-cn/docs/commands

主题

直接区官方主题库，看上自己中意的主题点进去就有使用教程
比如BlueLake主题：

  2. 修改根目录下_config.yml中theme的值为BlueLake，或者执行命令

官方主题库，详见https://hexo.io/themes

部署

在GitHub Pages上部署

官方部署文档，详见https://hexo.io/zh-cn/docs/github-pages

常见问题

博文摘要显示

  • 在正文中手动添加<!--more-->
  • 在文件开头代码块中设置excerpt来写文章摘要（优先级更高，摘要不会在正文中显示）
